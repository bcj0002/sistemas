Uno de los motivos principales de este aumento es el rápido y fácil acceso a tiendas online y la gran competencia que hay por el gran consumo de estos tiempos.
Cada vez más los jóvenes se interesan más por la ciberseguridad y en muchos casos prefieren usar ese conocimiento para destruir y no crear fácil acceso a internet.

¿Quiénes Suelen Ser los Ciberdelincuentes?

Cibercriminales individuales: son personas que actúan por su cuenta y riesgo y que buscan obtener beneficios económicos a través de sus acciones delictivas.
Insiders: son personas que trabajan dentro de una organización y que utilizan su posición privilegiada para llevar a cabo acciones delictivas.

Diferencias Sobre la Deep Web y la Dark Web

la Deep Web es una porción de Internet que no es indexada por los motores de búsqueda tradicionales como Google, ya sea porque están protegidos por contraseñas, son páginas web dinámicas que generan contenido de forma individualizada para cada usuario, o porque son simplemente sitios que no están enlazados desde otros sitios.
La Deep Web es esa parte de la web que no está indexada por los motores de búsqueda, mientras que la Dark Web es una porción de Internet dentro de la Deep web, que está oculta intencionalmente y que no es indexada por los motores de búsqueda.

Capas del Modelo OSI Implementadas por el SO (software) y Hardware

Las capas superiores como la capa 7 (Aplicación) es implementada por el sistema operativo (software) mientras que las capas inferiores como la capa 1 (Física ) son implementadas por el hardware.
Es importante mencionar que la implementación de las capas del modelo OSI no es una regla fija. En algunos casos, ciertas capas pueden ser implementadas por hardware o software, dependiendo de la configuración de la red y del dispositivo utilizado. Por ejemplo, en algunos dispositivos de red, la capa de enlace de datos puede ser implementada por hardware y software.

Qué es y Cómo Utilizar NMap

Es un sistema open source que sirve para escanear una red y sus puertos con el fin de recopilar información para completar tareas de seguridad.
Para utilizar Nmap, primero se debe entender su funcionamiento. Nmap envía paquetes sin procesar a los puertos del sistema para determinar el estado de los puertos. Luego de analizarlo, arroja las características de la red y un inventario completo del hardware y software que emplea . La respuesta que recibe Nmap contiene datos que permiten identificar los atributos de red y así crear un completo mapa de la red y un inventario del software y hardware conectado a la misma.

Qué es y Cómo Utilizar el Escaneo ARP

El protocolo de Resolución de Direcciones(address resolution protocol) es un protocolo de red común que permite que las redes traduzcan entre la dirección IP de un dispositivo y su dirección física de máquina.

El Escaneo de Puertos

es una técnica que se utiliza para descubrir los puertos abiertos en un sistema informático o dispositivo de red.
Los puertos son puntos de comunicación que permiten la transferencia de datos entre diferentes aplicaciones o servicios en un sistema.

Escaneo de Puertos TCP

El escaneo de puertos TCP es una técnica utilizada para descubrir los puertos abiertos en un sistema remoto. Se envía una solicitud de conexión a cada puerto en el rango deseado y se espera una respuesta. Si se recibe una respuesta, se sabe que el puerto está abierto y puede ser un posible punto de entrada para un ataque. El escaneo de puertos TCP puede ser realizado de varias formas, como por ejemplo mediante un escaneo SYN, en el que se envía una solicitud SYN-ACK para iniciar un proceso de conexión.

Escaneo de Puertos UDP

El escaneo de puertos UDP es similar al escaneo de puertos TCP pero se enfoca en los puertos UDP en lugar de los puertos TCP. Como UDP es un protocolo sin conexión, no hay un proceso de conexión como en TCP. En lugar de eso, se envía un paquete UDP a cada puerto y se espera una respuesta. Si se recibe una respuesta, se sabe que el puerto está abierto y puede ser un posible punto de entrada para un ataque.

Vulnerabilidades IDOR

Las vulnerabilidades IDOR (Insecure Direct Object Reference) son vulnerabilidades en aplicaciones web que permiten a un atacante acceder a recursos que no debería tener acceso, como información o funciones que pertenecen a otro usuario. Esto se debe a que la aplicación web utiliza identificadores internos para acceder a objetos en lugar de verificar si el usuario tiene permisos para acceder a ellos. Las vulnerabilidades IDOR pueden ser explotadas mediante la manipulación de los identificadores internos en las solicitudes.

SQL Injection

El SQL injection es un tipo de ataque cibernético que aprovecha las vulnerabilidades en las aplicaciones web que interactúan con bases de datos. Un atacante puede insertar código malicioso en una solicitud SQL enviada a la base de datos para obtener información confidencial o realizar acciones no autorizadas. Esto puede ocurrir cuando la aplicación web no valida o filtra adecuadamente los datos de entrada, permitiendo al atacante inyectar código malicioso en una solicitud SQL. El ataque SQL injection puede tener graves consecuencias para la seguridad de la aplicación web y de los datos almacenados en la base de datos.

Ataque XSS

El ataque XSS (Cross-Site Scripting) es una vulnerabilidad en aplicaciones web que permite a un atacante insertar scripts maliciosos en páginas web vistas por otros usuarios. Estos scripts se ejecutan en el navegador de las víctimas y pueden robar información sensible, realizar acciones no deseadas o comprometer la seguridad de la página.

Cómo Proteger una Red WIFI

Para proteger una red Wi-Fi, se pueden seguir algunas buenas prácticas:
 - Cambiar la contraseña del router regularmente y utilizar una contraseña segura y única.
 - Utilizar un cifrado fuerte, como WPA2 o WPA3, en lugar de WEP.
 - Desactivar la difusión del nombre de la red (SSID) para evitar que la red sea visible para los atacantes.
 - Filtrar las direcciones MAC para permitir solo dispositivos autorizados en la red.
 - Mantener el firmware del router actualizado con los últimos parches de seguridad.
 - Utilizar firewalls y software de seguridad en los dispositivos conectados a la red.

Hackeos por Ingeniería Social

Los ataques de ingeniería social se basan en manipular la psicología humana para obtener información confidencial o persuadir a las personas a realizar acciones no deseadas. Los hackers pueden utilizar técnicas como la suplantación de identidad, la creación de escenarios convincentes, el establecimiento de confianza y la explotación de la curiosidad o el miedo de las víctimas. Estos ataques pueden realizarse a través de llamadas telefónicas, correos electrónicos, mensajes de texto, redes sociales u otros medios de comunicación.

Autenticación Multifactor

La autenticación multifactor (MFA) es un método de verificación de identidad que requiere más de un factor para autenticar a un usuario. En lugar de depender únicamente de una contraseña, se utilizan múltiples factores, como una contraseña, un código único enviado a un dispositivo móvil, una huella dactilar o una tarjeta de acceso. La autenticación multifactor proporciona una capa adicional de seguridad, ya que incluso si un factor es comprometido, aún se requiere al menos otro factor para acceder a la cuenta o sistema.

Credencial Stuffing ¿Qué es?

El credential stuffing es un tipo de ataque cibernético en el que un atacante utiliza combinaciones de nombres de usuario y contraseñas robados de una fuente para intentar acceder a otras cuentas en diferentes servicios en línea. Los atacantes aprovechan el hecho de que muchas personas reutilizan las mismas credenciales en múltiples plataformas, lo que les permite probar esas credenciales en diferentes servicios y lograr acceso no autorizado a las cuentas de las víctimas.
